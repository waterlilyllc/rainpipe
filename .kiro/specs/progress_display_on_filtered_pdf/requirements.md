# Requirements Document

## Project Description (Input)
filtered_pdf実行時に進捗を画面に出す

## Requirements

### 1. Progress Visibility
**1.1** ユーザーは PDF 生成の各段階をリアルタイムで確認できる
**1.2** 各ステップの進捗状況（開始、処理中、完了）を表示する
**1.3** エラーが発生した場合、その旨と原因を表示する

### 2. Progress Information Display
**2.1** Raindrop.io からのブックマーク取得状況を表示
**2.2** キーワード・日付フィルタリング結果を表示
**2.3** Gatherly API による本文取得の進捗状況を表示（ポーリング回数、状態）
**2.4** GPT による要約生成の進捗状況を表示（現在何番目のブックマークか）
**2.5** PDF 生成の進捗状況を表示（ページ数、メモリ使用量など）
**2.6** メール送信の状況を表示（送信開始、成功/失敗）

### 3. Output Format
**3.1** ユーザーフレンドリーな形式で出力（emoji インジケータ使用）
**3.2** 構造化されたログで進捗の追跡が容易
**3.3** 進捗情報は 1 行、または複数行で適切に表示される
**3.4** タイムスタンプ情報は含める（オプション）

### 4. Non-Goals
- リアルタイム WebSocket ベースの Web UI 進捗表示（CLI のみ）
- 進捗情報の永続化・キャッシング
- Web API エンドポイントの追加
